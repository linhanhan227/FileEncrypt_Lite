# Secure File Encryptor

安全的文件加密工具，使用 AES-256-GCM + Argon2id 加密算法。

## 功能特性

- **军用级加密**：AES-256-GCM 对称加密 + Argon2id 密钥派生
- **自解密程序**：加密后生成可独立运行的 `.sfx.exe` 文件
- **一次性解密**：每个加密文件只能解密一次，无论成功失败
- **防暴力破解**：解密失败后程序自动删除，防止反复尝试
- **跨平台支持**：支持 Windows、macOS、Linux
- **进度条显示**：解密过程有可视化进度条
- **防重复运行**：检测并阻止多次运行同一加密文件

## 一次性解密机制

### 安全保证

无论以下何种情况发生，程序都保证只能运行一次：

| 场景 | 程序行为 |
|------|----------|
| 解密成功 | 文件释放，程序删除自身 |
| 密码错误 | 程序删除自身，无法再试 |
| 运行失败 | 程序删除自身，无法再试 |
| 重复运行 | 检测到已运行过，拒绝执行并删除 |

### 技术实现

1. **首次运行检测**：程序启动时在同目录创建隐藏标记文件 `.xxx.sfx.run`
2. **标记持久化**：无论程序运行结果如何，标记文件都会保留
3. **后续运行拒绝**：再次运行同一程序时，检测到标记文件存在，直接拒绝

## 加密算法

| 组件 | 算法 | 说明 |
|------|------|------|
| 对称加密 | AES-256-GCM | AEAD 加密模式 |
| 密钥派生 | Argon2id | 抗 GPU/ASIC 攻击 |
| 完整性验证 | HMAC-SHA256 | 数据完整性校验 |

## 使用方法

### 1. 加密文件

运行程序后输入要加密的文件路径，然后设置密码（8-32个字符）。

```bash
# 运行加密程序
./secure_file_encryptor.exe

# 按提示操作：
# 1. 输入要加密的文件路径
# 2. 设置密码（8-32字符）
# 3. 确认密码
# 4. 生成 .sfx.exe 文件
```

### 2. 解密文件

双击运行 `.sfx.exe` 文件，输入密码即可解密。

```
⚠ 重要警告：只有一次解密机会！
  • 输入正确的密码才能解密文件
  • 密码错误将永久无法恢复
  • 请务必牢记您的密码
```

### 3. 加密后程序行为

- 首次运行：要求输入密码，解密文件
- 再次运行：检测到重复运行，程序自动删除
- 密码错误：程序删除自身
- 解密成功：文件释放，程序删除自身

## 安全说明

### 为什么难以破解？

1. **Argon2id 密钥派生**
   - 需要大量内存（抗 GPU/ASIC）
   - 无法并行计算
   - 暴力破解成本极高

2. **AES-256-GCM**
   - 256 位密钥空间
   - 无已知破解方法
   - 军事级安全标准

### 密码安全建议

- 长度 ≥ 12 字符
- 包含大小写字母、数字、特殊字符
- 避免使用个人信息或常见词汇
